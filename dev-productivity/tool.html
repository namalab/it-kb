開発生産性

開発者が知っておくべきツールの使い分けと活用

はじめに

IT・特にシステム開発に携わる開発者向けに、ツールの使い分けに関するヒントをまとめておきました。
生産性向上の一助になれば幸いです。

対象者

  • システム開発に携わる開発者

得られる知識・ゴール

  • ツールの使い分けに関して一定の考え方が身につく
  • 結果として作業の生産性アップが期待できる

使用ツールの工夫

各種ツールの特徴を知り、うまく使い分けることで生産性向上に繋げることができます。
会社やお客様にツールを指定されてしまうこともありますが、自分で選択できる場合・提案できる場合には、是非下記の内容を参考に選択してください。

オフィス製品

ツール類の選択肢も多様化してきましたが、今でもマイクロソフトオフィス製品を活用する現場は多くあります。

資料作成に使える主な選択肢としては Word、Excel、PowerPoint がありますが、割と多くの現場において「Excel しか使わない」とか「PowerPoint しか使わない」といった文化があります。
それぞれ特徴は全く異なりますので、それぞれの特性を踏まえて適宜選択することで、生産性向上に繋げることが可能です。

オフィス製品（Word）

メリット
  • 目次作成や索引を簡単に作成可能
  • 見出しに章番号（1-1、1-2、…）を割り当てることができて非常に便利
デメリット
  • 用紙サイズ以上の大きな図や表を書くことができない
向き不向き
  • 目次や章番号を付けるような長い文書では最適
    • マニュアル、手順書など
  • 何ページにも及ぶ印刷文書にも最適
  • 大きな図や表を埋め込むのには向かない
    →これらが必要な場合は Excel で作成のうえ「別紙」として添付した方が良い

オフィス製品（Excel）

メリット
  • 本来の表計算ソフトとして、表やグラフを作成したりデータを蓄積する用途には最適
  • 表を書くだけでなく、文書や作図にも使える汎用性
  • 大きな図や表を書いても「ページに収まるよう印刷」することも可
デメリット
  • 章番号は手動で書く必要があり、多くの現場・多くの資料で更新漏れ等が散見される
  • 印刷ズレ
向き不向き
  • 1ファイルに複数シート＋Excel方眼の構成により、汎用的な資料作成ツールとして優秀
    （ここでは、Excel方眼の是非については触れません）
  • ただし印刷ズレが酷いため、印刷前提なら使うべきではない
    • 印刷の改ページを意識してページの枠線を入れているようなフォーマットをたまに見かけるが、生産性の面では最低
  • 章番号のようなものも手動でメンテするのは生産性の面で無駄
  • セル結合は生産性低下の元凶。行列の切り取りや選択もやりにくくなるため絶対禁止
    • Excelのバージョンによって、結合セルを含んだ計算結果が異なることもある
      https://www.itmedia.co.jp/news/articles/2201/21/news144.html
    • 見た目上でセンタリングしたいだけならセルの書式設定＞配置＞横位置を「選択範囲内で中央」にすれば良い
    • Excel方眼に列サイズの異なる表を埋め込みたいなら、別シートに表を書いて「リンクされた図」で貼りつければ良い

オフィス製品（PowerPoint）

メリット
  • 見た目を良くするための機能が多数
  • 特にスマートアート機能ではプレゼンでよく使われるイメージを簡単に挿入可能
デメリット
  • 章番号は手動で書く必要あり
  • ページからはみ出した範囲は表示されない
向き不向き
  • 当たり前だが「プレゼン」資料には最適
    • インパクトと説得力を与えることができる
    • 「この提案を顧客に納得してもらって決断してもらう」といった場合には絶対 PowerPoint
    • セミナーや勉強会のように説明に用いる資料でも、1ページずつ見せて説明できるので向いている
  • ただし、ページからはみ出した範囲は見えなくなるため、長文や細かい情報も含めて正確に伝えたい資料には向かない
    • 1ページに収めることや見た目を優先して、本来書くべきことを省略したり表現を変えてしまう人が多い
    • 要件定義書や設計書、マニュアル類を PowerPoint で書くプロジェクトもあるが、お勧めしない
  • PowerPoint で資料を書こうとすると、意外と長時間かかることが多いので注意

専用ツールの活用

専用ツールを使うことで、作図の手間を軽減したり、各種便利機能が使えるなど、生産性の面で非常に高い効果があります。

代表的なものを記しますので、参考になさってください。

ER図

  • ERMaster（Eclipseプラグイン）
    • 綺麗な図を簡単に描ける
    • DB からのリバースや DDL の出力が可能
    • Eclipse ベースしかないのが残念だが、使い勝手の良いソフト
  • A5:SQL Mk-2
    • SQLクライアントツール
    • DB からのリバースや DDL の出力が可能
    • ER図の機能としては少々癖がある
  • MySQL WorkBench
    • MySQL 専用だが優れた SQLクライアントツール
    • DB からのリバースや DDL の出力が可能
    • ER図の機能も優秀
  • SI ObjectBrowser ER（有償）
    • 有償のER図作成ツール

クラス図

  • astah* UML（有償）
    • 各種UMLを作画するための専用ツール
    • ソースコードからクラス図をリバース可能

ガントチャート

  • Microsoft Project（有償）
    • 定番のプロジェクト管理ツール
    • WBS をガントチャートとして管理できるだけでなく、要員やコストの管理も可能
  • Redmine
    • 広く使われているチケット管理ツール
    • ガントチャート表示もできる

汎用だけど便利な Draw.io

Draw.io という作画ツールがあります。
こちらは何らかの図に特化した専用ツールではないものの、作図ツールとしてはかなり使いやすく洗練されており、綺麗な図をパパッと作成できるので時短になります。
システム構成図を Excel で頑張って描くよりは、こちらを使って手早く仕上げましょう。

  • 汎用的な作図ツール
  • フローチャートや AWS 構成図等を書くためのアイコンが標準で多数用意されている
  • ブラウザ上で作図できるだけでなく、VSCode 拡張機能としてローカルファイルも作成可
  • 作成した図は、クリップボードにコピーしたり画像ファイルとして出力することができる
  • VSCode にて拡張子を「～.drawio.svg」とすれば、そのまま SVG ファイルとして外部から読み込むことが可能（画像ファイルとして Export する手間が不要）

汎用だけど便利な VisualStudio Code

Microsoft の VisualStudio Code というテキストエディタがあります。VSCode と省略されることもあります。
テキストエディタと言いつつ、各種拡張機能にも対応し、様々なプログラミング言語のヒント表示やデバッグにも対応する高機能なソフトウェアです。

前述の Draw.io の拡張機能もあります。

マークダウンの活用

こちらはツールではありませんが、「マークダウン」というテキストフォーマットがあります。
簡単な記述で HTML のタグのような機能を表現できるものです。
メモやちょっとした文書を書くのに向いています。

  • テキストエディタで書いて整形
  • HTML や Word、PDF に変換できるのは勿論、marp 等のツールを用いてスライドも出力可能
  • 「中身＞見た目」の資料作りにおいて、まずマークダウンで箇条書き→整理してから清書するとスムーズ
